# YYC3-UI 智能检测系统 v2.0

## 📋 系统概述

完整的自动化检测、修复和提交系统，确保代码质量并在提交远程前进行全面验证。

## 🚀 快速开始

### 手动检测命令

```bash
# 快速检查（核心功能）
pnpm check:quick

# 完整检查（推荐）
pnpm check:full

# CI/CD 模拟检查
pnpm check:ci

# 自动修复问题
pnpm check:fix

# 验证文件和配置
pnpm check:verify

# 模拟 pre-push 流程
pnpm check:push
```

### 自动触发

推送代码到远程时自动激活：

```bash
git push origin main
```

系统会自动：
1. 执行全维度检测
2. 发现问题自动修复
3. 修复后重新检测
4. 通过后才推送

## 🔧 自动修复机制

### 支持的自动修复

1. **ESLint 修复**
   ```bash
   pnpm lint:fix
   ```
   自动修复代码规范问题

2. **Prettier 格式化**
   ```bash
   pnpm format:write
   ```
   自动格式化代码

3. **核心包构建**
   ```bash
   pnpm --filter=yyc3-ui build
   ```
   清理并重新构建

4. **依赖安装**
   ```bash
   pnpm install --frozen-lockfile=false
   ```
   检查并安装缺失依赖

### 修复流程

1. 检测失败
2. 记录失败项
3. 执行对应修复命令
4. 验证修复结果
5. 重新检测（最多 2 次）
6. 生成修复报告

## 📊 配置文件说明

### `.ci-config.json`

完整 CI/CD 配置文件，支持：

- ✅ 自定义检查项
- ✅ 超时设置
- ✅ 自动修复命令
- ✅ 跳过规则
- ✅ 通知配置
- ✅ CI 平台配置

### 配置示例

```json
{
  "checks": {
    "critical": [
      {
        "name": "ESLint",
        "command": "pnpm lint",
        "timeout": 300,
        "autoFix": "pnpm lint:fix"
      }
    ]
  },
  "skip": {
    "files": ["**/*.test.ts"],
    "errors": ["TS2589", "TS2345"]
  },
  "autoFix": {
    "enabled": true,
    "maxRetries": 2
  }
}
```

1. 编辑 `.ci-config.json`
2. 在对应级别添加检查项
3. 更新本文档

### 添加自动修复

1. 在检查项中添加 `autoFix` 字段
