## 🔄 系统工作流程

```
用户执行: git push
    ↓
自动触发: .husky/pre-push
    ↓
执行: scripts/pre-push-full.sh
    ↓
┌─────────────────────────────────┐
│ 阶段1: 快速检查                  │
│ - Git 仓库状态                   │
│ - 核心构建产物                    │
│ - 依赖安装完整性                  │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│ 阶段2: 代码质量检查              │
│ - ESLint 代码规范                │
│ - TypeScript 类型检查            │
│ - Prettier 格式检查              │
│ - 核心包构建                     │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│ 阶段3: CI/CD 检查                │
│ - 依赖安全审计                   │
│ - Registry 配置验证              │
│ - 单元测试                       │
└────────────┬────────────────────┘
             ↓
        检测结果
             │
    ┌────────┴────────┐
    ↓                 ↓
  通过              失败
    │                 │
    ↓                 ↓
 自动推送      自动修复
                   │
                   ↓
              重新检测
              (最多2次)
                   │
           ┌───────┴───────┐
           ↓               ↓
         通过            失败
           │               │
           ↓               ↓
        自动推送       中止推送
```

## 🎯 核心特性

### ✅ 全维度检测

- **关键检查**: ESLint, 核心构建, Git 状态
- **重要检查**: TypeScript, 单元测试, 格式检查
- **可选检查**: 安全审计, Registry, 文档完整性
- **CI 检查**: 18 个检测项，覆盖 CI/CD 所需

### ✅ 智能修复

- 自动修复 ESLint 问题
- 自动格式化代码
- 自动重新构建
- 自动安装依赖
- 最多重试 2 次

### ✅ 零人工干预

- Git push 自动触发
- 检测失败自动修复
- 修复后自动重试
- 通过后自动推送

### ✅ 详细反馈

- 彩色输出
- 三阶段报告
- 错误详情
- 修复建议

## 🚀 可用命令

### 手动检测命令

```bash
pnpm check:quick   # 快速检查 (核心项)
pnpm check:full    # 完整检查 (10项)
pnpm check:ci      # CI/CD 检查 (18项)
pnpm check:fix     # 自动修复
pnpm check:push    # 模拟推送检查
pnpm check:verify  # 快速验证
```

### 自动检测

```bash
git push origin main  # 自动触发全流程
```
