# 提交前智能检查系统

## 📋 概述

YYC3-UI 配备了智能的提交前检查系统，在每次 `git commit` 前自动执行代码质量检查，失败则自动修复，确保代码质量。

## 🔄 工作流程

```
git commit 触发
    ↓
pre-commit hook 激活
    ↓
运行智能检查脚本
    ├─ ESLint 代码规范检查
    ├─ Prettier 格式检查
    └─ TypeScript 类型检查
    ↓
检查结果
├─ ✓ 通过 → 允许提交
└─ ✗ 失败 → 自动修复
              ↓
          重新检查（最多2次）
              ↓
          ├─ ✓ 通过 → 允许提交
          └─ ✗ 失败 → 阻止提交，提示手动修复
```

## 🎯 检查项目

### 关键检查（失败会阻止提交）

1. **ESLint 代码规范**
   - 检查代码风格和潜在错误
   - 自动修复：运行 `pnpm lint:fix`

2. **Prettier 格式检查**
   - 检查代码格式
   - 自动修复：运行 `pnpm format:write`

### 重要检查（失败会警告）

3. **TypeScript 类型检查**
   - 检查类型错误
   - 需要手动修复

## 🛠️ 手动使用

```bash
# 手动运行提交前检查
pnpm check:commit

# 或直接运行脚本
./scripts/pre-commit-check.sh
```

## ⚙️ 配置说明

### Husky Hook

位置：`.husky/pre-commit`

```bash
#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

# 运行提交前智能检查
./scripts/pre-commit-check.sh
```

### 自动修复机制

- **最大重试次数**: 2 次
- **自动修复项目**:
  - ESLint 问题
  - Prettier 格式问题
- **暂存修复**: 自动将修复后的文件添加到暂存区

## 🔧 自定义配置

### 跳过检查

如果需要临时跳过检查（不推荐）：

```bash
# 方法1: 使用 --no-verify
git commit --no-verify -m "your message"

# 方法2: 设置环境变量
HUSKY=0 git commit -m "your message"
```

### 调整检查项

编辑 `scripts/pre-commit-check.sh` 中的检查项：

```bash
# 添加新检查
run_check "新检查项" \
    "your-check-command" \
    true  # true=关键检查, false=非关键
```

## 🚀 最佳实践

1. **提交前预览**: 在提交前先运行 `pnpm check:commit` 查看检查结果
2. **及时修复**: 看到错误提示后，优先使用自动修复，失败再手动修复
3. **保持代码质量**: 不要频繁使用 `--no-verify` 跳过检查
4. **阅读报告**: 仔细阅读检查报告，了解问题所在

## 🆘 常见问题

### Q: 如何禁用某个检查项？

A: 编辑 `scripts/pre-commit-check.sh`，注释掉对应的 `run_check` 调用

### Q: 自动修复后为什么还是失败？

A: TypeScript 类型错误通常需要手动修复，请查看详细错误信息

### Q: 如何添加自定义检查项？

A: 在 `scripts/pre-commit-check.sh` 中添加新的 `run_check` 函数调用
